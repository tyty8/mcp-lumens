## ¿Qué es MCP?

El Model Context Protocol (MCP) es un estándar abierto desarrollado por Anthropic que define cómo los clientes de IA se comunican con herramientas externas y fuentes de datos. Piense en él como el USB-C de las integraciones de IA: un conector único y universal que funciona entre cualquier cliente compatible y cualquier servidor compatible, independientemente de quién los haya construido.

Antes de MCP, cada integración de herramientas de IA era una implementación única: código personalizado para cada combinación de modelo de IA y servicio externo. MCP reemplaza esa fragmentación con un protocolo estandarizado. Una vez que un servidor implementa MCP, cualquier cliente compatible con MCP puede usarlo de inmediato, sin código de pegamento ni adaptadores personalizados.

## Por qué MCP es importante

- **Cualquier cliente, cualquier servidor.** Un servidor escrito una sola vez funciona con Claude Desktop, Cursor, su agente personalizado o cualquier otro host MCP.
- **Independiente del proveedor.** El protocolo es abierto y no está vinculado a ningún proveedor de IA en particular. Los servidores pueden utilizarse con diferentes modelos a medida que el ecosistema crece.
- **Estándar abierto.** La especificación es pública. Cualquiera puede construir servidores, clientes o herramientas alrededor de ella, creando un ecosistema compartido en lugar de integraciones aisladas.

## Requisitos previos

- [Claude Desktop](https://claude.ai/download) (u otro cliente compatible con MCP)
- Node.js 18 o superior (ejecute `node --version` para verificar)
- Una terminal o símbolo del sistema

## Conectar su primer servidor MCP

### Paso 1: Instalar Claude Desktop

Descargue e instale Claude Desktop desde [claude.ai/download](https://claude.ai/download). Inicie sesión con su cuenta de Anthropic.

### Paso 2: Abrir el archivo de configuración

Claude Desktop lee la configuración del servidor MCP desde un archivo JSON en disco.

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

Abra ese archivo en un editor de texto. Si aún no existe, créelo.

### Paso 3: Agregar un servidor MCP

Esta es una configuración mínima que agrega el Servidor MCP de Sistema de Archivos, con alcance a su directorio de inicio:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname"
      ]
    }
  }
}
```

Reemplace `/Users/yourname` con el directorio al que desea que tenga acceso la IA. En Windows use una ruta como `C:\\Users\\yourname`.

Para agregar múltiples servidores, añada más entradas bajo `mcpServers`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/yourname"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_yourTokenHere"
      }
    }
  }
}
```

### Paso 4: Reiniciar Claude Desktop

Cierre completamente y vuelva a abrir Claude Desktop. La aplicación lee el archivo de configuración al iniciarse, por lo que los cambios solo tienen efecto después de un reinicio.

### Paso 5: Probar la conexión

Inicie una nueva conversación y pídale a Claude algo que requiera el servidor. Por ejemplo, con el servidor de sistema de archivos:

> "Lista los archivos en mi carpeta Documentos."

Si el servidor está conectado correctamente, Claude invocará la herramienta y devolverá resultados reales. Verá un indicador de uso de herramienta en la conversación.

## Solución de problemas

- **"No hay herramientas disponibles"** — Verifique que el archivo de configuración sea JSON válido (sin comas finales, comillas correctas). Use un validador de JSON si no está seguro.
- **El servidor no arranca** — Ejecute el comando de instalación manualmente en una terminal (`npx -y @modelcontextprotocol/server-filesystem /tmp`) para confirmar que Node.js y npx funcionan correctamente.
- **Errores de permisos en macOS** — Es posible que Claude Desktop necesite Acceso Completo al Disco. Vaya a Configuración del Sistema > Privacidad y Seguridad > Acceso Completo al Disco y agregue Claude.
- **Variables de entorno no establecidas** — Verifique que las claves `env` en el archivo de configuración coincidan exactamente con lo que especifica la documentación del servidor.
