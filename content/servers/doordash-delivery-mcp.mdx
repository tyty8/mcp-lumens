## Descripción general

El servidor DoorDash Delivery MCP conecta directamente con la API de DoorDash Drive, la plataforma de entrega a domicilio que permite a restaurantes, dark kitchens, tiendas de alimentación y cualquier negocio de comida usar la flota de repartidores de DoorDash sin necesidad de tener repartidores propios. A través de este servidor, el asistente de IA puede gestionar de forma conversacional todo el ciclo de vida de un pedido de delivery, desde su creación hasta la confirmación de entrega en el domicilio del cliente.

La integración facilita especialmente la gestión de múltiples pedidos simultáneos en horas punta, que es precisamente cuando más difícil resulta mantener el control de cada envío. El asistente puede consultar en tiempo real el estado de todos los pedidos activos, identificar cuáles están retrasados, comunicar al cliente una actualización de tiempo estimado de entrega y gestionar incidencias como repartidores que no llegan a recoger el pedido o entregas fallidas, todo sin que el personal del restaurante tenga que dejar de atender la operación.

Para las dark kitchens, que operan exclusivamente para delivery sin sala de restaurante, este servidor es especialmente crítico porque su negocio entero depende de que las entregas se produzcan de forma fluida y puntual. La capacidad de monitorear automáticamente cada pedido y recibir alertas sobre cualquier incidencia permite gestionar operaciones de alto volumen con equipos reducidos, uno de los principales desafíos de este modelo de negocio donde la eficiencia operativa directamente impacta en la rentabilidad.

## Casos de uso para negocios

- **Restaurantes con servicio a domicilio:** Creación automática de pedidos de delivery en DoorDash cuando llega un pedido al sistema de punto de venta, sin necesidad de introducir manualmente los datos del cliente y la dirección de entrega.
- **Dark kitchens de alta rotación:** Monitoreo simultáneo de 20-30 pedidos activos en hora punta del viernes noche, con alertas inmediatas si algún repartidor no recoge el pedido en el tiempo establecido.
- **Cadenas de restauración:** Seguimiento consolidado del rendimiento de delivery por local — tiempo medio de preparación, tiempo medio de entrega, valoraciones y tasa de incidencias por establecimiento.
- **Tiendas de alimentación y supermercados:** Gestión de pedidos de compra rápida a domicilio mediante la flota de DoorDash, compitiendo con los grandes operadores de quick commerce sin necesidad de flota propia.
- **Pastelerías y negocios de catering:** Programación de recogidas futuras para pedidos especiales (tartas de cumpleaños, catering de evento) con confirmación automática al cliente.
- **Negocios de comida de conveniencia:** Análisis de las métricas de delivery semanales para identificar los productos más pedidos a domicilio y ajustar la oferta del menú de delivery en consecuencia.

## Configuración básica

Registra tu negocio como merchant en DoorDash Drive para obtener acceso a la API y las credenciales necesarias:

```bash
npx @jordandalton/doordash-mcp-server
```

Configura las credenciales en tu cliente MCP:

```json
{
  "mcpServers": {
    "doordash": {
      "command": "npx",
      "args": ["@jordandalton/doordash-mcp-server"],
      "env": {
        "DOORDASH_DEVELOPER_ID": "tu-developer-id",
        "DOORDASH_KEY_ID": "tu-key-id",
        "DOORDASH_SIGNING_SECRET": "tu-signing-secret",
        "DOORDASH_ENVIRONMENT": "production"
      }
    }
  }
}
```

Las credenciales de la API de DoorDash Drive se obtienen desde el portal de desarrolladores en developer.doordash.com. Para empezar, puedes usar el entorno sandbox (`"DOORDASH_ENVIRONMENT": "sandbox"`) para probar la integración sin crear pedidos reales.
