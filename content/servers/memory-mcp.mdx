## Overview

The Memory MCP Server solves one of the fundamental limitations of language models: the context window resets between conversations. By maintaining a persistent knowledge graph stored on disk, this server lets an AI client remember facts, relationships, and observations across sessions. Entities (people, projects, concepts) and the relations between them are stored as a structured graph and retrieved on demand.

This server is especially valuable for building AI assistants that accumulate knowledge over time — remembering your preferences, tracking the state of long-running projects, or maintaining a shared knowledge base for a team. The knowledge graph is stored as a JSON file on disk, making it easy to inspect, backup, and version control.

## Configuration

By default the knowledge graph is stored in a `memory.json` file in the server's working directory. You can customize the storage path with the `MEMORY_FILE_PATH` environment variable.

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"],
      "env": {
        "MEMORY_FILE_PATH": "/Users/yourname/.claude/memory.json"
      }
    }
  }
}
```

## Key Capabilities

- Create, update, and delete named entities with typed attributes
- Store relations between entities (e.g. "Alice works-at Acme Corp")
- Add and remove observations (timestamped facts) on any entity
- Search entities by name or type
- Retrieve the full graph or subgraphs around a specific entity
- Persistent storage survives process restarts and new conversation sessions
