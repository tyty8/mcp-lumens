## Overview

The Filesystem MCP Server exposes your local file system to AI clients in a controlled, auditable way. You specify one or more root directories at startup, and the server restricts all operations to those paths — the AI cannot read or write outside the boundaries you define. This makes it practical for project-level AI assistance without exposing your entire disk.

Common use cases include letting an AI read your codebase to answer questions, write new files, refactor existing code across multiple files, or generate configuration files in the right locations. Combined with a capable AI client, this server effectively gives you an AI pair programmer with direct file access.

## Configuration

Pass one or more allowed directory paths as arguments when launching the server. All paths must be absolute.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/yourname/projects/my-app",
        "/Users/yourname/documents"
      ]
    }
  }
}
```

## Key Capabilities

- Read file contents (text and binary)
- Write and overwrite files
- List directory contents recursively
- Create and delete directories
- Move and rename files
- Search file contents with pattern matching
- All operations are sandboxed to the configured root paths
